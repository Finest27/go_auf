# ThePressUSA Auto-Journalist Bot - Կոդի Քարտեզ (Codebase Map)

Այս փաստաթուղթը տրամադրում է բոտի ամբողջական կառուցվածքը, ֆունկցիաների տեղաբաշխումը և տվյալների բազայի կառուցվածքը։ Ծրագիրը գրված է Go (Golang) լեզվով՝ Clean Architecture սկզբունքներով։

## 1. Պանակների և Ֆայլերի Կառուցվածք (Directory Structure)

### `cmd/bot/`
- **`main.go`**: Ծրագրի գլխավոր մուտքի կետը։ Սկզբնավորում է բոլոր կախվածությունները (database, configs, repositories, usecases) և գործարկում է վեբ սերվերը ու բոտի հիմնական աշխատանքային ցիկլերը։

### `internal/config/`
- **`config.go`**: Պարունակում է ծրագրի կարգավորումների (Config) կառուցվածքը։ Պատասխանատու է բազայի `app_settings` աղյուսակից կարգավորումները բեռնելու (`Load()`, `Get()`) և պահպանելու (`Save()`) համար։

### `internal/domain/`
Այստեղ սահմանված են ծրագրի հիմնական մոդելներն ու ինտերֆեյսները։
- **`models/`**: Պարունակում է `Article`, `Feed` և այլ հիմնական տվյալների կառուցվածքները։
- **`repository/`**: `ArticleRepository` և `FeedRepository` ինտերֆեյսներն են, որոնք սահմանում են տվյալների բազայի հետ աշխատանքի կանոնները։
- **`services/`**: Սահմանում է `AIProvider` ինտերֆեյսը AI մոդելների հետ աշխատելու համար։

### `internal/usecase/`
Այստեղ է գտնվում բիզնես տրամաբանությունը (Core Logic)։
- **`article_usecase.go`**: Պարունակում է `ArticleUseCase` կառուցվածքը։ Հիմնական ֆունկցիաներն են.
  - `ExecuteScrapeCycle`: Հավաքագրում է նորությունները RSS աղբյուրներից, պահպանում բազայում և ուղարկում AI-ին վերամշակման (`processWithAI`)։
  - `ExecutePublishCycle`: Ստուգում է վերամշակված հոդվածները և հերթով հրապարակում դրանք։
  - `PublishSingle`: Մեկ հոդվածի հրապարակում WordPress-ում։
- **`article_usecase_test.go`**: Պարունակում է բիզնես տրամաբանության թեստերը։

### `internal/infra/`
Ենթակառուցվածքային մոդուլներ՝ արտաքին գործիքների և ինտեգրացիաների համար։

#### `infra/database/`
- **`sqlite.go`**: Ստեղծում է կապ SQLite բազայի հետ (`NewSQLiteDB`)։ Պարունակում է բազայի ստեղծման հրամանները (schema) և միգրացիաները։

#### `infra/repository/`
- **`sqlite_article_repo.go`**: Իրականացնում է բազայի հետ աշխատանքի հարցումները՝ պահպանել, թարմացնել, ջնջել և ստանալ հոդվածները (CRUD ֆունկցիաներ)։

#### `infra/api/`
- **`server.go`**: Գործարկում է Fiber վեբ սերվեր (Live UI վահանակ)։ Սահմանում է API երթուղիները (routes)՝ բոտը միացնելու/անջատելու, հերթը ստուգելու, կարգավորումները փոխելու և WebSocket-ի միջոցով log-եր փոխանցելու համար։
- **`static/`**: Պարունակում է HTML, CSS և JS ֆայլեր կառավարման վահանակի (Live UI) համար։

#### `infra/ai/`
- **`nvidia.go`**: Ներառում է `NvidiaProvider`՝ Nvidia NIM LLM-ի հետ աշխատելու տրամաբանությունը։
- **`modelslab.go`**: Ներառում է `ModelsLabProvider` նկարների գեներացման կամ վերամշակման համար։
- **`factory.go`**: Ֆունկցիա (`NewProvider`) AI մոդելի ճիշտ պրովայդերը ընտրելու համար։

#### `infra/publisher/`
- **`wordpress.go`**: Ներառում է `WPClient`՝ WordPress-ի REST API-ի միջոցով հոդվածներ և մեդիա ֆայլեր հրապարակելու համար (`Publish`, `UploadImageFromURL`)։
- **`linker.go`**: Ներառում է `InternalLinker`՝ SEO-ի համար ներքին հղումներ ինքնաշխատ ավելացնելու ֆունկցիա։

#### `infra/scraper/`
- **`universal.go`**: Հոդվածների քրոուլինգ (տեքստի և նկարների հավաքագրում) որոշակի հղումներից։
- **`rss.go`**: RSS ֆիդերի կարդացման տրամաբանություն (`FetchRSSLinks`)։
- **`stealth.go`**: Go-rod բրաուզերի աննկատ աշխատանքի ֆունկցիաներ (`NewStealthBrowser`)։

#### `infra/utils/`
- **`logger.go`**: Վարում է log-երը և ուղարկում WebSockets-ով (Live UI)։
- **`downloader.go`**: Ֆայլերի (օր.՝ նկարների) ներբեռնման անվտանգ ֆունկցիաներ։
- **`sanitizer.go`**: HTML տեքստի մաքրում՝ անվտանգության համար։
- **`browser.go`**: Օգնող ֆունկցիաներ բրաուզերի համար։


## 2. Տվյալների Բազայի Կառուցվածք (Database Schema)
Բոտն օգտագործում է **SQLite** բազա (`bot_ultimate.db` ֆայլը)։ Այն ունի 3 հիմնական աղյուսակ.

1. **`app_settings`**:
   - Պահում է համակարգի կարգավորումները (միշտ ունի `id = 1`):
   - Դաշտեր՝ `wp_url`, `wp_username`, `wp_app_password`, `nvidia_api_key`, և այլ ժամային (interval) կարգավորումներ ու AI prompt-եր։

2. **`rss_topics`**:
   - Պահում է RSS աղբյուրների ցանկը։
   - Դաշտեր՝ `id`, `name`, `wp_category_id` (WordPress-ում որ բաժնում հրապարակել), `rss_url`։

3. **`articles`**:
   - Հիմնական աշխատանքային աղյուսակն է, որտեղ պահվում են բոլոր հոդվածները։
   - Հիմնական դաշտեր՝
     - `source_url`: Օրիգինալ հոդվածի հղումը (UNIQUE)։
     - `title`, `content`: Բնօրինակ վերնագիր և տեքստ։
     - `rewritten_content`: AI-ի կողմից մշակված տեքստը։
     - `image_url`, `image_alt`: Նկարի հղումը և նկարագրությունը։
     - `status`: Հոդվածի կարգավիճակը (pending, rewritten, published, failed)։
     - `retry_count`, `next_retry_at`: Սխալների դեպքում կրկնակի փորձերի կառավարում։

## 3. Հիմնական Ֆունկցիաներ և Աշխատանքի Տրամաբանություն (Core Workflow)

1. **Սկզբնավորում (main.go):**
   Բացվում է SQLite բազան, կարդացվում են կարգավորումները, սկզբնավորվում են պրովայդերները և գործարկվում է Fiber վեբ սերվերը `8080` պորտով։

2. **Scraping (Հավաքագրում):**
   - `ExecuteScrapeCycle` ֆունկցիան (usecase պանակում) պարբերաբար անցնում է `rss_topics`-ի միջով։
   - Կանչում է `scraper.FetchRSSLinks`՝ հղումներ ստանալու համար։
   - Եթե հղումը նոր է, `universal.Scrape` ֆունկցիայով քաշում է հոդվածը։
   - Պահպանում է `articles` աղյուսակում `"pending"` կարգավիճակով։

3. **Վերամշակում (AI Rewriting):**
   - `processWithAI` ֆունկցիան օգտագործում է `NvidiaProvider` AI մոդելը հոդվածը վերափոխելու, SEO նկարագրություն և նկարի Alt տեքստ գեներացնելու համար։
   - Հաջողության դեպքում կարգավիճակը դառնում է `"rewritten"`։

4. **Հրապարակում (Publishing):**
   - `ExecutePublishCycle` ֆունկցիան ընտրում է պատրաստի (`rewritten`) հոդվածներ և կանչում է `PublishSingle`։
   - `WPClient`-ը նախ ներբեռնում և WordPress-ի Media Library է ուղարկում նկարը (`UploadImageFromURL`), ապա ստեղծում է հոդվածը (`Publish`):
   - Հրապարակված հոդվածների կարգավիճակը փոխվում է `"published"`։

5. **Կառավարման վահանակ (Live UI):**
   - `internal/infra/api/server.go` ֆայլում սահմանված են վեբ-ինտերֆեյսի ուղիները։ Բոտի աշխատանքի տվյալներն ուղարկվում են էկրան իրական ժամանակում (WebSockets-ի և `logger.go`-ի միջոցով)։
