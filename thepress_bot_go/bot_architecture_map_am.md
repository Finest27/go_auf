# ThePress Bot Ultimate Ճարտարապետության Քարտեզ

Այս փաստաթուղթը տրամադրում է բոտի կոդի բազայի համապարփակ քարտեզ՝ հեշտացնելով նավարկումը և փոփոխությունների կիրառումը: Այն նկարագրում է թղթապանակների կառուցվածքը, հիմնական ֆայլերը, դրանց գործառույթները և տվյալների բազայի սխեման (սխեման):

## Թղթապանակների Կառուցվածք

*   **`cmd/bot/`**: Պարունակում է հավելվածի մուտքի կետը (entry point):
*   **`internal/config/`**: Կառավարում է հավելվածի կարգավորումները:
*   **`internal/domain/`**: Սահմանում է հիմնական էությունները (մոդելներ) և պայմանագրերը (ինտերֆեյսներ):
*   **`internal/infra/`**: Պարունակում է կոնկրետ իրականացումներ (տվյալների բազա, արտաքին API-ներ, scraper-ներ):
*   **`internal/usecase/`**: Իրականացնում է հիմնական բիզնես տրամաբանությունը:

## Հիմնական Ֆայլեր և Ֆունկցիաներ

### 1. `cmd/bot/main.go`
Սա հավելվածի մեկնարկային կետն է: Այն նախաստեղծում (initialize) է տվյալների բազան, բեռնում է կարգավորումները, տեղադրում է կախվածությունները և սկսում է ֆոնային ցիկլերը:

*   **`main()`**: Նախաստեղծում է տվյալների բազան, կախվածությունները, սերվերը և գործարկում է բոտի աշխատանքը:
*   **`BotRunner.Start()`**: Սկսում է ֆոնային scraping-ի և հրապարակման ցիկլերը (`runScrapeLoop`, `runPublishLoop`):
*   **`BotRunner.Stop()`**: Կանգնեցնում է ֆոնային ցիկլերը:
*   **`runScrapeLoop(ctx)`**: Պարբերաբար գործարկում է `ArticleUseCase.ExecuteScrapeCycle()`:
*   **`runPublishLoop(ctx)`**: Պարբերաբար գործարկում է `ArticleUseCase.ExecutePublishCycle()`:

### 2. `internal/config/config.go`
Կառավարում է հավելվածի կարգավորումները, որոնք հիմնականում պահվում են SQLite տվյալների բազայում, բայց կարող են նաև կարդացվել JSON-ից:

*   **`Load()`**: Բեռնում է կարգավորումները տվյալների բազայից հիշողության մեջ:
*   **`Save()`**: Պահպանում է ընթացիկ հիշողության մեջ եղած կարգավորումները տվյալների բազայում:
*   **`Get()`**: Վերադարձնում է ընթացիկ կարգավորումների օբյեկտը:

### 3. `internal/domain/models/`
Սահմանում է հավելվածում օգտագործվող հիմնական տվյալների կառուցվածքները:

*   **`article.go`**: Սահմանում է `Article` կառուցվածքը (SourceURL, Title, Content, RewrittenContent, Status, PublishDate, slug, tags, meta description և այլն):

### 4. `internal/usecase/article_usecase.go`
Պարունակում է կենտրոնական բիզնես տրամաբանությունը, որը կազմակերպում է (orchestrates) scraping-ի, վերագրման և հրապարակման գործընթացները:

*   **`ArticleUseCase.ExecuteScrapeCycle()`**: Անցնում է կազմաձևված RSS թեմաների վրայով, գործարկում է scraper-ը և պահպանում նոր հոդվածները տվյալների բազայում «pending» կարգավիճակով:
*   **`ArticleUseCase.ExecutePublishCycle()`**: Գտնում է սպասող հոդվածներ, գործարկում է AI-ին տեքստը վերագրելու համար, այնուհետև գործարկում է հրապարակիչին՝ WordPress-ում գրառում ստեղծելու համար: Թարմացնում է հոդվածի կարգավիճակը՝ դարձնելով «published»:

### 5. `internal/infra/database/sqlite.go` (և հարակից ֆայլեր)
Կառավարում է SQLite տվյալների բազայի կապը և սխեման:

*   **`NewSQLiteDB(path)`**: Բացում է կապ դեպի SQLite տվյալների բազա:

### 6. `internal/infra/` (Ենթաթղթապանակներ)
Պարունակում է արտաքին փոխազդեցությունների իրականացումները:

*   **`api/`**: Վեբ սերվերի տրամաբանություն և HTTP մշակիչներ (handlers) բոտի կառավարման վահանակի (dashboard/control panel) համար:
*   **`repository/`**: SQL իրականացումներ՝ հոդվածները և կարգավորումները տվյալների բազայում պահպանելու և վերցնելու համար:
*   **`scraper/`**: Տրամաբանություն RSS feed-երից XML ստանալու և այն վերլուծելու (parse) համար:
*   **`publisher/`**: Տրամաբանություն WordPress REST API-ի հետ նույնականացման (authentication) և փոխազդեցության համար՝ գրառումներ ստեղծելու նպատակով:
*   **`ai/`**: AI պրովայդերների (ինչպես Nvidia կամ ModelsLab) հետ հաղորդակցվելու իրականացումներ՝ հոդվածի տեքստը վերագրելու համար:

## Տվյալների Բազայի Սխեմա (SQLite)

Բոտը օգտագործում է SQLite տվյալների բազա (սովորաբար `bot_ultimate.db`) հետևյալ հիմնական աղյուսակներով.

1.  **`app_settings`**:
    *   Պահպանում է բոտի գլոբալ կարգավորումները:
    *   Օրինակներ. Գործարկման միջակայքերը, WordPress-ի տվյալները (URL, օգտանուն, հավելվածի գաղտնաբառ), AI API բանալիները և վերագրման համար օգտագործվող հուշումները (prompts):
2.  **`rss_topics`**:
    *   Պահպանում է scraping արվող RSS feed-երի ցանկը:
    *   Կապում է յուրաքանչյուր feed URL որոշակի WordPress Category ID-ի հետ:
3.  **`articles`**:
    *   Պահպանում է բոլոր հավաքագրված և մշակված հոդվածները:
    *   Հիմնական սյուներ (columns).
        *   `source_url` (Հիմնական նույնացուցիչ՝ կրկնօրինակներից խուսափելու համար)
        *   `title` (Բնօրինակ վերնագիր)
        *   `content` (Բնօրինակ բովանդակություն)
        *   `rewritten_content` (AI-ի կողմից ստեղծված բովանդակություն)
        *   `status` (օրինակ՝ "pending", "published", "failed")
        *   `publish_date`
        *   SEO մետատվյալներ (`slug`, `tags`, `meta_description`)
        *   `wp_post_id` (WordPress-ում ստեղծված գրառման ID-ն)

## Ինչպես Անել Փոփոխություններ

*   **Նոր կարգավորման տարբերակ ավելացնելիս**: Թարմացրեք `internal/config/config.go`, `app_settings` աղյուսակի սխեման (եթե ավելացնում եք նոր սյունակ) և հնարավոր է `internal/infra/api` մշակիչները՝ թույլ տալով փոխել այն վահանակի միջոցով:
*   **Scraping-ի տրամաբանությունը փոփոխելիս**: Նայեք `internal/infra/scraper/`:
*   **Հոդվածների վերագրման եղանակը փոխելիս**: Թարմացրեք AI ինտեգրումը `internal/infra/ai/`-ում և հուշումների տրամաբանությունը `internal/usecase/article_usecase.go`-ում:
*   **WordPress-ի հրապարակումը փոխելիս**: Փոփոխեք կոդը `internal/infra/publisher/`-ում:
*   **Նոր ֆոնային առաջադրանք ավելացնելիս**: Ավելացրեք նոր ցիկլային ֆունկցիա `cmd/bot/main.go`-ում `BotRunner`-ի ներսում և համապատասխան մեթոդ համապատասխան use case-ում:
