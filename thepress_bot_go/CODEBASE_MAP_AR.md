# ThePress Bot Ultimate Ճարտարապետական Քարտեզ

Այս փաստաթուղթը ներկայացնում է բոտի կոդի համապարփակ քարտեզը՝ հեշտացնելով նավարկումը և փոփոխությունների կիրառումը: Այն ուրվագծում է թղթապանակների կառուցվածքը, հիմնական ֆայլերը, դրանց գործառույթները և տվյալների բազայի սխեման:

## Թղթապանակների Կառուցվածքը (Directory Structure)

*   **`cmd/bot/`**: Պարունակում է հավելվածի մուտքի կետը (entry point):
*   **`internal/config/`**: Կառավարում է հավելվածի կարգավորումները:
*   **`internal/domain/`**: Սահմանում է հիմնական էությունները (մոդելներ) և պայմանագրերը (ինտերֆեյսներ):
*   **`internal/infra/`**: Պարունակում է կոնկրետ իրականացումներ (տվյալների բազա, արտաքին API-ներ, scraper-ներ):
*   **`internal/usecase/`**: Իրականացնում է հիմնական բիզնես տրամաբանությունը:

## Հիմնական Ֆայլեր և Ֆունկցիաներ

### 1. `cmd/bot/main.go`
Սա հավելվածի մեկնարկային կետն է: Այն նախաստեղծում է տվյալների բազան, բեռնում կարգավորումները, կարգավորում կախվածությունները և սկսում ֆոնային ցիկլերը (loops):

*   **`main()`**: Նախաստեղծում է տվյալների բազան, կախվածությունները, սերվերը և սկսում bot runner-ը:
*   **`BotRunner.Start()`**: Սկսում է ֆոնային scraping և publishing ցիկլերը (`runScrapeLoop`, `runPublishLoop`, `runAILoop`):
*   **`BotRunner.Stop()`**: Կանգնեցնում է ֆոնային ցիկլերը:
*   **`runScrapeLoop(ctx)`**: Պարբերաբար կատարում է `ArticleUseCase.ExecuteScrapeCycle()`:
*   **`runPublishLoop(ctx)`**: Պարբերաբար կատարում է `ArticleUseCase.ExecutePublishCycle()`:
*   **`runAILoop(ctx)`**: Պարբերաբար կատարում է `ArticleUseCase.ExecuteAICycle()`:

### 2. `internal/config/config.go`
Կառավարում է հավելվածի կարգավորումները, որոնք հիմնականում պահվում են SQLite տվյալների բազայում, բայց կարող են վերադառնալ JSON-ի՝ որպես պահեստային տարբերակ:

*   **`Load()`**: Բեռնում է կարգավորումները տվյալների բազայից հիշողության մեջ:
*   **`Save()`**: Պահպանում է ընթացիկ հիշողության մեջ եղած կարգավորումները հետ՝ տվյալների բազայում:
*   **`Get()`**: Վերադարձնում է ընթացիկ կոնֆիգուրացիայի օբյեկտը:

### 3. `internal/domain/models/`
Սահմանում է հիմնական տվյալների կառուցվածքները, որոնք օգտագործվում են ողջ հավելվածում:

*   **`article.go`**: Սահմանում է `Article` struct-ը (SourceURL, Title, Content, RewrittenContent, Status, PublishDate, slug, tags, meta description և այլն):
*   **`feed.go`**: Սահմանում է `Feed` struct-ը:

### 4. `internal/usecase/article_usecase.go`
Պարունակում է կենտրոնական բիզնես տրամաբանությունը, որը ղեկավարում է քերելու (scraping), վերաշարադրելու (rewriting) և հրապարակելու (publishing) գործընթացները:

*   **`ArticleUseCase.ExecuteScrapeCycle()`**: Անցնում է կազմաձևված RSS թեմաներով, գործարկում scraper-ը և պահպանում նոր հոդվածները տվյալների բազայում «pending» (սպասող) կարգավիճակով:
*   **`ArticleUseCase.ExecuteAICycle()`**: Մշակում է «pending» հոդվածները AI-ի միջոցով (Nvidia կամ ModelsLab):
*   **`ArticleUseCase.ExecutePublishCycle()`**: Գտնում է հրապարակման ենթակա հոդվածները և գործարկում publisher-ը՝ WordPress-ում տեղադրելու համար: Թարմացնում է հոդվածի կարգավիճակը «published» (հրապարակված):

### 5. `internal/infra/database/sqlite.go` (և հարակից ֆայլեր)
Սպասարկում է SQLite տվյալների բազայի միացումը և սխեման:

*   **`NewSQLiteDB(path)`**: Բացում է միացում SQLite տվյալների բազային: Օգտագործում է `modernc.org/sqlite` գրադարանը (CGO-free):

### 6. `internal/infra/` (Ենթաթղթապանակներ)
Պարունակում է արտաքին փոխազդեցությունների իրականացումները:

*   **`api/`**: Վեբ սերվերի տրամաբանություն և HTTP handler-ներ բոտի վահանակի/կառավարման վահանակի (dashboard) համար: (Օր. `server.go`)
*   **`repository/`**: SQL իրականացումներ՝ տվյալների բազայից հոդվածներ և կարգավորումներ պահպանելու/ստանալու համար: (Օր. `sqlite_article_repo.go`)
*   **`scraper/`**: RSS ալիքներից XML ստանալու, վերլուծելու և կայքերից տեքստեր քերելու տրամաբանություն (օգտագործում է `go-rod` գրադարանը JS կայքերի համար):
*   **`publisher/`**: WordPress REST API-ի հետ նույնականանալու և հաղորդակցվելու տրամաբանություն՝ փոստեր ստեղծելու համար: (Օր. `wordpress.go`, `linker.go`)
*   **`ai/`**: AI պրովայդերների հետ (ինչպիսիք են Nvidia-ն կամ ModelsLab-ը) հաղորդակցվելու իրականացումներ՝ հոդվածի տեքստը վերաշարադրելու համար: (Օր. `nvidia.go`, `modelslab.go`)
*   **`utils/`**: Օգնական ֆունկցիաներ (logger, browser և այլն):

## Տվյալների Բազայի Սխեմա (SQLite Database Schema)

Բոտն օգտագործում է SQLite տվյալների բազա (սովորաբար `bot_ultimate.db`) հետևյալ հիմնական աղյուսակներով.

1.  **`app_settings`**:
    *   Պահպանում է բոտի գլոբալ կոնֆիգուրացիան:
    *   Օրինակներ. Գործարկման միջակայքեր, WordPress հավատարմագրեր (URL, username, app password), AI API բանալիներ և վերաշարադրման համար օգտագործվող հուշումներ (prompts):
2.  **`rss_topics`**:
    *   Պահպանում է RSS ալիքների ցանկը քերելու (scraping) համար:
    *   Կապում է յուրաքանչյուր ալիքի URL-ը համապատասխան WordPress Category ID-ի հետ:
3.  **`articles`**:
    *   Պահպանում է բոլոր քերված և մշակված հոդվածները:
    *   Հիմնական սյուներ.
        *   `source_url` (Հիմնական նույնացուցիչ՝ կրկնօրինակներից խուսափելու համար)
        *   `title` (Բնօրինակ վերնագիր)
        *   `content` (Բնօրինակ բովանդակություն)
        *   `rewritten_content` (AI-ի կողմից ստեղծված բովանդակություն)
        *   `status` (օրինակ՝ "pending", "published", "failed")
        *   `publish_date`
        *   SEO մետատվյալներ (`slug`, `tags`, `meta_description`)
        *   `wp_post_id` (Ստեղծված հոդվածի ID-ն WordPress-ում)
