# ThePress Bot Ultimate Ճարտարապետական Քարտեզ

Այս փաստաթուղթը ներկայացնում է բոտի կոդային բազայի համապարփակ քարտեզը, ինչը հեշտացնում է կոդի նավիգացիան և փոփոխությունների կիրառումը: Այն նկարագրում է թղթապանակների կառուցվածքը, հիմնական ֆայլերը, նրանց գործառույթները և տվյալների բազայի (DB) սխեման:

## Տեխնոլոգիական Ստեկ

*   **Ծրագրավորման լեզու:** Go (Golang)
*   **Վեբ Շրջանակ (Web Framework):** Fiber (`github.com/gofiber/fiber/v2`)՝ վեբ սերվերի, REST API-ի և WebSocket-ների համար:
*   **Տվյալների բազա:** SQLite (`bot_ultimate.db`), օգտագործելով `github.com/jmoiron/sqlx` և `modernc.org/sqlite` գրադարանները:
*   **Վեբ Քերում (Scraping):** Բրաուզերի վրա հիմնված քերում `github.com/go-rod/rod` գրադարանով և RSS հոսքերի վերլուծություն `github.com/mmcdole/gofeed` գրադարանով:
*   **Արհեստական Բանականություն (AI):** Ինտեգրում Nvidia NIM և ModelsLab գործիքների հետ՝ բովանդակության և նկարների մշակման համար:
*   **Հրապարակում:** Ինտեգրում WordPress REST API-ի հետ:
*   **Ճարտարապետական Մոդել:** Clean Architecture (Մաքուր Ճարտարապետություն)՝ խիստ բաժանված domain (մոդելներ/ինտերֆեյսներ), usecase (բիզնես տրամաբանություն) և infra (ադապտերներ, սքրեյփերներ, հրատարակիչներ, տվյալների բազա) շերտերի:
*   **Կատարման Մոդել:** Մոնոլիտ օրկեստրացիայի ձևաչափ (Monolithic orchestration pattern), որտեղ քերումը, AI մշակումը և հրապարակումը կատարվում են մեկ կատարման ցիկլում (execution loop):

## Թղթապանակների Կառուցվածք

*   **`cmd/bot/`**: Պարունակում է հավելվածի մուտքի կետը (entry point):
*   **`internal/config/`**: Կառավարում է հավելվածի կարգավորումները:
*   **`internal/domain/`**: Սահմանում է հիմնական էությունները (մոդելները) և պայմանագրերը (ինտերֆեյսները):
*   **`internal/infra/`**: Պարունակում է կոնկրետ իրականացումներ (տվյալների բազա, արտաքին API-ներ, սքրեյփերներ):
*   **`internal/usecase/`**: Իրականացնում է հիմնական բիզնես տրամաբանությունը:

## Հիմնական Ֆայլեր և Գործառույթներ

### 1. `cmd/bot/main.go`
Սա հավելվածի սկզբնակետն է: Այն նախաստեղծում է տվյալների բազան, բեռնում կարգավորումները, տեղադրում կախվածությունները և սկսում ֆոնային գործընթացները:

*   **`main()`**: Նախաստեղծում է DB-ն, կախվածությունները, սերվերը և սկսում բոտի աշխատանքը:
*   **`BotRunner.Start()`**: Սկսում է ֆոնային քերման և հրապարակման ցիկլերը (`runScrapeLoop`, `runAILoop`, `runPublishLoop`):
*   **`BotRunner.Stop()`**: Դադարեցնում է ֆոնային ցիկլերը:
*   **`runScrapeLoop(ctx)`**: Պարբերաբար գործարկում է `ArticleUseCase.ExecuteScrapeCycle()`:
*   **`runAILoop(ctx)`**: Պարբերաբար գործարկում է AI մշակման ցիկլը:
*   **`runPublishLoop(ctx)`**: Պարբերաբար գործարկում է `ArticleUseCase.ExecutePublishCycle()`:

### 2. `internal/config/config.go`
Կառավարում է հավելվածի կարգավորումները, որոնք հիմնականում պահվում են SQLite տվյալների բազայում:

*   **`Load()`**: Բեռնում է կարգավորումները տվյալների բազայից հիշողության մեջ:
*   **`Save()`**: Պահպանում է ընթացիկ հիշողության մեջ եղած կարգավորումները տվյալների բազայում:
*   **`Get()`**: Վերադարձնում է ընթացիկ կոնֆիգուրացիայի օբյեկտը:

### 3. `internal/domain/models/`
Սահմանում է հիմնական տվյալների կառույցները, որոնք օգտագործվում են հավելվածի ողջ ընթացքում:

*   **`article.go`**: Սահմանում է `Article` ստրուկտուրան (SourceURL, Title, Content, RewrittenContent, Status, PublishDate, slug, tags, meta description և այլն):
*   **`feed.go`**: RSS հոսքերի հետ կապված մոդելներ:

### 4. `internal/usecase/article_usecase.go`
Պարունակում է կենտրոնական բիզնես տրամաբանություն, որը կառավարում է քերման, վերագրման (rewrite) և հրապարակման գործընթացները:

*   **`ArticleUseCase.ExecuteScrapeCycle()`**: Անցնում է կազմաձևված RSS թեմաների վրայով, գործարկում սքրեյփերը և պահպանում նոր հոդվածները տվյալների բազայում "pending" (սպասող) կարգավիճակով:
*   **`ArticleUseCase.ExecutePublishCycle()`**: Գտնում է սպասող հոդվածները, գործարկում AI-ն բովանդակությունը վերագրելու համար և ապա գործարկում հրատարակիչին WordPress-ում տեղադրելու համար: Թարմացնում է հոդվածի կարգավիճակը դառնալով "published" (հրապարակված):

### 5. `internal/infra/database/sqlite.go` (և հարակից ֆայլեր)
Կառավարում է SQLite տվյալների բազայի միացումը և սխեման:

*   **`NewSQLiteDB(path)`**: Բացում է միացում դեպի SQLite տվյալների բազա:

### 6. `internal/infra/` (Ենթաթղթապանակներ)
Պարունակում են արտաքին փոխազդեցությունների իրականացումները:

*   **`api/`**: Վեբ սերվերի տրամաբանություն և HTTP մշակիչներ (handlers) բոտի կառավարման վահանակի (dashboard) համար:
*   **`repository/`**: SQL իրականացումներ հոդվածները և կարգավորումները տվյալների բազայում պահպանելու/ստանալու համար:
*   **`scraper/`**: Տրամաբանություն RSS հոսքերից XML ստանալու և վերլուծելու համար:
*   **`publisher/`**: Տրամաբանություն WordPress REST API-ի միջոցով նույնականացվելու և գրառումներ ստեղծելու համար (`internal/infra/publisher/wordpress.go`):
*   **`ai/`**: Իրականացումներ AI մատակարարների (օրինակ՝ Nvidia կամ ModelsLab) հետ հաղորդակցվելու համար՝ հոդվածի տեքստը վերագրելու նպատակով:
*   **`utils/`**: Օժանդակ ֆունկցիաներ (լոգեր, բրաուզեր բացող, և այլն):

## Տվյալների Բազայի Սխեմա (SQLite)

Բոտն օգտագործում է SQLite տվյալների բազա (սովորաբար `bot_ultimate.db`), որը պարունակում է հետևյալ հիմնական աղյուսակները.

1.  **`app_settings`**:
    *   Պահպանում է բոտի գլոբալ կարգավորումները:
    *   Օրինակներ. Աշխատանքի ինտերվալներ, WordPress-ի տվյալներ (URL, username, app password), AI API բանալիներ և վերագրման համար օգտագործվող հուշումներ (prompts):
2.  **`rss_topics`**:
    *   Պահպանում է RSS հոսքերի ցանկը, որոնք պետք է քերվեն:
    *   Յուրաքանչյուր հոսքի URL-ը կապում է կոնկրետ WordPress Կատեգորիայի ID-ի հետ:
3.  **`feeds`**:
    *   Լրացուցիչ աղյուսակ հոսքերի համար (URL, անուն, վերջին ստուգում, ակտիվության կարգավիճակ):
4.  **`articles`**:
    *   Պահպանում է բոլոր քերված և մշակված հոդվածները:
    *   Հիմնական սյունակներ.
        *   `id` (Բանալի)
        *   `source_url` (Հիմնական նույնացուցիչ՝ կրկնօրինակներից խուսափելու համար)
        *   `title` (Բնօրինակ վերնագիր)
        *   `content` (Բնօրինակ բովանդակություն)
        *   `rewritten_content` (AI-ի կողմից ստեղծված բովանդակություն)
        *   `image_url`, `image_alt`
        *   `status` (օր.՝ "pending", "published", "failed")
        *   `publish_date`
        *   SEO մետատվյալներ (`slug`, `tags`, `meta_description`, `focus_keywords`)
        *   `category_id`, `category`
        *   `created_at`
        *   `retry_count`, `next_retry_at`

## Ինչպես Կատարել Փոփոխություններ

*   **Նոր կարգավորման ավելացում**: Թարմացրեք `internal/config/config.go`-ն, `app_settings` աղյուսակի սխեման (եթե ավելացնում եք նոր սյունակ) և հավանաբար `internal/infra/api` մշակիչները (handlers)՝ կառավարման վահանակից այն փոխելու հնարավորություն տալու համար:
*   **Քերման տրամաբանության փոփոխում**: Նայեք `internal/infra/scraper/` թղթապանակում գտնվող ֆայլերին:
*   **Հոդվածների վերագրման ձևի փոփոխում**: Թարմացրեք AI ինտեգրումը `internal/infra/ai/`-ում և հուշումների (prompt) տրամաբանությունը `internal/usecase/article_usecase.go`-ում:
*   **WordPress հրապարակման փոփոխում**: Փոփոխեք կոդը `internal/infra/publisher/`-ում:
*   **Նոր ֆոնային առաջադրանքի ավելացում**: Ավելացրեք նոր ցիկլային ֆունկցիա `cmd/bot/main.go`-ում `BotRunner`-ի ներսում և համապատասխան մեթոդ համապատասխան use case-ում:
*   **Անվտանգություն և ստանդարտներ**: Համոզվեք, որ օգտագործում եք Go ստանդարտ գրադարանները և գոյություն ունեցող կախվածությունները նորերի փոխարեն, ինչպես նաև պահպանում եք անվտանգ կոնկուրենտություն և CGO-ից զերծ սահմաններ:
