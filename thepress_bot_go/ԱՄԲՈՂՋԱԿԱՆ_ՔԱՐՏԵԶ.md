# ThePress Bot Ultimate - Ամբողջական Ճարտարապետական Քարտեզ

Այս փաստաթուղթը ներկայացնում է «ThePressUSA Auto-Journalist Bot»-ի կոդի, տվյալների բազայի և աշխատանքի հոսքի ամբողջական քարտեզը հայերեն լեզվով։ Բոտը գրված է Go (Golang) ծրագրավորման լեզվով և հետևում է Clean Architecture (Մաքուր Ճարտարապետություն) սկզբունքներին։

## 1. Դիրեկտորիաների Կառուցվածք (Directory Structure)

Նախագծի հիմնական դիրեկտորիան `thepress_bot_go`-ն է:

*   **`cmd/bot/`**: Ծրագրի մուտքի կետը: `main.go` ֆայլը սկզբնավորում է բազան, կարգավորումները, միացնում է կախվածությունները, սերվերը և սկսում է ֆոնային պրոցեսները (Scrape, AI, Publish):
*   **`internal/config/`**: Ծրագրի կարգավորումների կառավարում (`config.go`): Կարգավորումները պահվում են SQLite բազայում (`app_settings` աղյուսակ):
*   **`internal/domain/`**: Հիմնական էնթիթիները և ինտերֆեյսները (Մոդելներ):
    *   `models/article.go`: `Article` մոդելը (հոդվածի տվյալներ):
    *   `models/feed.go`: `Feed` մոդելը:
    *   `repository/interfaces.go`: Տվյալների բազայի ինտերֆեյսները:
    *   `services/interfaces.go`, `ai_interface.go`: Scraper, Publisher, և AI ծառայությունների ինտերֆեյսները:
*   **`internal/usecase/`**: Բիզնես տրամաբանությունը (`article_usecase.go`): Պարունակում է 3 հիմնական ցիկլերի (cycles) իրականացումը:
*   **`internal/infra/`**: Արտաքին համակարգերի և բազայի հետ կապող շերտ:
    *   **`ai/`**: AI ծառայությունների ադապտերներ (Nvidia `nvidia.go`, ModelsLab `modelslab.go`):
    *   **`api/`**: Վեբ սերվերի իրականացում Fiber ֆրեյմվորքով (`server.go`) և UI-ի ստատիկ ֆայլեր (`static/`):
    *   **`database/`**: SQLite բազայի միացման կարգավորումներ (`sqlite.go`):
    *   **`publisher/`**: WordPress-ում հրապարակելու տրամաբանություն (`wordpress.go`) և ներքին հղումների ավելացում (`linker.go`):
    *   **`repository/`**: Տվյալների բազայի հարցումների իրականացում (`sqlite_article_repo.go`):
    *   **`scraper/`**: RSS ֆիդերի կարդալու (`rss.go`) և կայքերից հոդվածների աննկատ (stealth) քերելու տրամաբանություն (`universal.go`, `stealth.go`): Օգտագործում է `go-rod` և `gofeed`։
    *   **`utils/`**: Օգնող ֆունկցիաներ (լոգեր, websocket hub, բրաուզեր, sanitizer):

## 2. Տվյալների Բազա (SQLite Database)

Բոտն օգտագործում է SQLite բազա, ֆայլի անվանումն է `bot_ultimate.db`: Բազան պարունակում է հետևյալ հիմնական աղյուսակները.

1.  **`app_settings`**:
    *   Պահում է բոտի հիմնական կարգավորումները:
    *   Ներառում է WordPress-ի մուտքային տվյալներ (URL, username, app password), AI API բանալիներ (Nvidia, ModelsLab), System Prompt-եր, և աշխատանքի ինտերվալներ:
2.  **`rss_topics`**:
    *   Պահում է RSS ֆիդերի ցանկը:
    *   Կապում է յուրաքանչյուր RSS հասցեն (URL) WordPress-ի համապատասխան կատեգորիայի ID-ի հետ:
3.  **`articles`**:
    *   Հիմնական աղյուսակն է, որտեղ պահվում են քերված և մշակված հոդվածները:
    *   Դաշտեր՝ `id`, `source_url` (եզակի), `title`, `content` (օրիգինալ բովանդակություն), `rewritten_content` (AI-ի կողմից վերաշարադրված տեքստ), `image_url`, SEO տվյալներ (`slug`, `tags`, `meta_description`, `focus_keywords`), `status` (pending, rewritten, published, failed), `category_id`, և այլն:
4.  **`feeds`**:
    *   Պահում է ֆիդերի մասին ընդհանուր ինֆորմացիա (url, name, last_check, active):

## 3. Աշխատանքի Հոսք (Workflow) և Ֆոնային Պրոցեսներ

Բոտի աշխատանքը հիմնված է 3 զուգահեռ (concurrent) գործող ցիկլերի (loops) վրա, որոնք ղեկավարվում են `BotRunner`-ի կողմից (`cmd/bot/main.go`):

### Ա. Քերելու Ցիկլ (Scrape Loop)
*   **Ֆունկցիա:** `runScrapeLoop` -> `ArticleUseCase.ExecuteScrapeCycle()`
*   **Նկարագրություն:** Բոտը պարբերաբար անցնում է բոլոր կարգավորված RSS թեմաներով:
*   **Ընթացք:**
    1.  Կարդում է RSS ֆիդը և ստանում հոդվածների հղումները:
    2.  Ստուգում է, արդյոք հղումն արդեն կա `articles` աղյուսակում: Եթե ոչ՝ շարունակում է:
    3.  Stealth բրաուզերի միջոցով (շրջանցելով Cloudflare և այլ պաշտպանություններ) բացում է հղումը և քերում է հոդվածի բովանդակությունն ու նկարը:
    4.  Մաքրում է HTML-ը և պահպանում բազայում **`pending`** (սպասող) կարգավիճակով:

### Բ. AI Մշակման Ցիկլ (AI Loop)
*   **Ֆունկցիա:** `runAILoop` -> `ArticleUseCase.ExecuteAICycle()`
*   **Նկարագրություն:** Վերցնում է `pending` կամ ձախողված (`failed`) հոդվածները և վերաշարադրում դրանք հայերենով:
*   **Ընթացք:**
    1.  Գտնում է 1 չմշակված հոդված:
    2.  Ուղարկում է Nvidia API-ին (Multi-Stage Rewriting Pipeline):
        *   **Stage 1:** Հանում է հիմնական փաստերը:
        *   **Stage 2:** Վերաշարադրում է փաստերը հայերենով՝ պրոֆեսիոնալ լրագրողական ոճով և վերադարձնում խիստ JSON ֆորմատով (վերնագիր, բովանդակություն, SEO նկարագրություն, keywords, slug):
    3.  Եթե JSON-ը սխալ է, բոտն ավտոմատ պահանջում է ուղղել այն (self-correction):
    4.  Հաջողության դեպքում հոդվածի կարգավիճակը դառնում է **`rewritten`** (վերաշարադրված): Ձախողման դեպքում՝ **`failed`** և ավելացվում է retry-ի ժամանակ:

### Գ. Հրապարակման Ցիկլ (Publish Loop)
*   **Ֆունկցիա:** `runPublishLoop` -> `ArticleUseCase.ExecutePublishCycle()`
*   **Նկարագրություն:** Վերցնում է մշակված հոդվածները և հրապարակում WordPress-ում:
*   **Ընթացք:**
    1.  Գտնում է **`rewritten`** կարգավիճակով հոդված:
    2.  **Internal Linking:** Տեքստի վերջում ավելացնում է 3 հղում նախկինում հրապարակված նմանատիպ հոդվածների վրա (`linker.go`):
    3.  **Image Processing:**
        *   Ներբեռնում է գլխավոր նկարը (Featured Image):
        *   Եթե միացված է ModelsLab-ը կամ AI նկարի մշակումը, մաքրում է նկարը (լոգոների և տեքստերի հեռացում):
        *   Վերբեռնում է նկարը WordPress Media Library:
        *   Ներբեռնում, վերբեռնում և փոխարինում է տեքստի մեջ առկա մյուս (inline) նկարները:
    4.  Օգտագործելով WordPress REST API-ն՝ հրապարակում է հոդվածը ճիշտ կատեգորիայում և SEO տվյալներով:
    5.  Հաջողության դեպքում կարգավիճակը դառնում է **`published`** (հրապարակված):

## 4. Լրացուցիչ Բաղադրիչներ

*   **Վեբ Ինտերֆեյս (UI):** Fiber վեբ սերվերը ապահովում է կառավարման վահանակ (Control Panel), որտեղից կարելի է փոխել կարգավորումները, ավելացնել RSS թեմաներ, տեսնել հերթը (queue), խմբագրել և ձեռքով հրապարակել հոդվածները:
*   **Իրական Ժամանակի Լոգեր:** WebSocket-ների միջոցով UI-ում ցուցադրվում են բոտի աշխատանքի լոգերը իրական ժամանակում (`utils.BroadcastLog`):
